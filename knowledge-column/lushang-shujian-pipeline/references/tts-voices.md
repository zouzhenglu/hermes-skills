# Edge TTS 中文语音参考

## 经验法则

> **默认用 `zh-CN-YunxiNeural`，0% 语速。不要加速。不要用 Yunyang。**

Yunxi（讲故事风格）是中文讲书的最佳选择。温暖自然，自带叙述节奏。Yunyang（专业播音腔）太像新闻联播，不适合陪伴型内容。

## 安装

Edge TTS 免费，无需 API Key，不限调用次数。

```bash
# Debian/Ubuntu（需要 --break-system-packages 绕过 PEP 668）
pip3 install --break-system-packages edge-tts
```

## 语速教训（重要）

**初始设定**：YunyangNeural +10%（275 汉字/分钟）
**用户反馈**：「最开始那几句话，有点突然太快了」
**修正**：YunxiNeural +0%（250 汉字/分钟）

原因：+10% 加速让开头像抢跑，没有呼吸感。通勤场景听众要的是陪伴，不是效率。0% 让 TTS 句间自然停顿。

## 时长估算

基于 0% 语速，250 汉字/分钟：

| 期类型 | 目标时长 | 所需汉字 |
|--------|---------|---------|
| 引读 | 8 分钟 | ~2000 |
| 核心期 | 10 分钟 | ~2500 |
| 总结 | 15 分钟 | ~3750 |

## 脚本写作适配

配合 Yunxi 写脚本的要点：
- **短句为主**。句号多，逗号少。TTS 靠句号自然停顿。
- **段落间空一行** → TTS 读到这里会停一拍。
- **避免 30 字以上的单句** → TTS 会一口气念完，听着累。
- **避免多层嵌套复句**（「虽然……但是……因为……所以……而且……」）→ TTS 语调会平得像念经。
- **写完自己默读一遍**，喘不过气就断句。
