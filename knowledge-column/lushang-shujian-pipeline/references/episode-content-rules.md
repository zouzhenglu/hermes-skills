# 分集文章内容规范

## 用户明确要求

分集文章（ep01-ep07）**不是播客 show notes**。不能只有时间轴和金句。
必须是**可以独立阅读的图文内容**——用户不看音频也能获得完整价值。

## 正确做法

每期文章 = 脚本内容改写为阅读格式，包含：

- 场景钩子 → 文章第一段
- 核心观点的完整展开 → 多个段落，带小标题
- 书中的故事和案例 → 完整写出
- 关键引用 → 用金句卡片突出
- 思考题/练习 → 暖色提示框

## 错误做法

- ❌ 只放时间轴（00:00 xxx / 03:00 yyy）
- ❌ 只放金句 + 预告
- ❌ 「详见音频第X分钟」
- ❌ 纯 `<p>` 段落堆砌，无视觉层次

## 必须使用的模板组件

详见 `references/article-layout.md`。每篇文章必须至少用到：
- 章节标题（border-left 左边框）
- 金句卡片（深色 background）
- 至少一个暖色提示框（bg_warm）
- 如有对比数据，用表格

## 音频嵌入说明

微信 API **不支持**自动嵌入音频到文章（`mpvoice` 标签被拦截，`<audio>` 标签被过滤）。
文章顶部放音频入口提示：告知用户去后台编辑器 → 插入音频 → 从素材库选择。
素材库中音频文件命名规则：`{book_id}_{书名}_ep{NN}_{标签}.mp3`
