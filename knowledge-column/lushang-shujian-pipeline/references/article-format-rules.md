# 文章格式规则

## 核心原则

文章做音频做不到的事。音频擅长讲故事和情绪，文章擅长表格、清单、框架、对比。**文章不是音频的文字版，是音频的使用说明书。**

## 不要放的东西

- ❌ 「阅读指南」「这不是音频的文字版」等元说明文字（用户反馈：多余，删掉）
- ❌ 音频里讲过的故事原文（一笔带过即可）
- ❌ 空洞的过渡语（「接下来我们来聊聊」「值得注意的是」）
- ❌ AI 腔结尾（「希望对你有所帮助」「让我们一起成长」）

## 必须放的东西

- ✅ **至少一张表格**（概念对比 / 步骤清单 / 层级诊断）
- ✅ **全书骨架速览**（树形结构，30 秒看懂）
- ✅ **行动指南**（可截图执行，每步有时间预估）
- ✅ **底部标注**：合集标签 + 原创声明 + AI辅助生成声明
- ✅ **评论区互动问题**

## WeChat 安全 HTML 规范

微信编辑器兼容的 HTML 是有限的。以下 CSS 会被过滤或导致乱码：

**✅ 可用：**
- `color`, `font-size`, `font-weight`, `text-align`
- `background`, `background-color`（纯色）
- `padding`, `margin`
- `border`, `border-left`, `border-radius`
- `max-width`（限制 640px 为手机宽度）

**❌ 不可用（会被过滤/乱码）：**
- `background: linear-gradient(...)` — 渐变会变纯色
- `box-shadow` — 阴影消失
- `display: flex`, `grid` — 布局崩塌
- `position: absolute/relative` — 不可靠
- `@import`, `<link>`, `<style>` 块 — 全部过滤
- `font-family` 里的系统字体栈 — 会被替换

**降级策略**：用纯色块 + 表格 + 边框替代复杂 CSS。如果用户要看视觉效果，用 browser 工具截图预览。

## 音频文件命名

```
{book_id}_{书名}_ep{NN}_{标签}.mp3
```

示例：`book-001_断舍离_ep01_引读.mp3`

好处：本地文件一目了然，上传到微信素材库后也能看到书名。
